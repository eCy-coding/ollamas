For reliable `write_host_file` writes to the Desktop artifact dir, first (re)start the bridge with that dir in its write roots:
```
BRIDGE_WRITE_ROOTS="$PWD:/tmp/llm-bridge:$HOME/.llm-mission-control:$HOME/Desktop/ollamas-automator" bash bin/host-bridge/start-bridge.sh
```
Then run `./node_modules/.bin/tsx orchestration/bin/automator-probe.ts`. It reads the live model list (`ollama list`, embedding models skipped), and SEQUENTIALLY, for each model: makes `~/Desktop/ollamas-automator/<model>/`, dispatches (via `scripts/agent-dispatch.mjs` → POST /api/agent/chat, default `OLLAMAS_URL=http://127.0.0.1:3000`) the task "author Automator-compatible artifacts supporting ollamas (start-server shell, cockpit AppleScript, /api curl, README) in that dir", then **scans the dir** to record what was actually produced. A model counts as "produced" when it wrote ≥1 real file (independent of its VERDICT). Flags: `--dry` (plan only), `--models a,b`, `--steps N` (default 6), `--json`. Reports which model produced which artifacts (the tracking matrix).

**`--task daily`** (vO36): the task becomes a DAILY, RECURRING, SUSTAINABLE automation (a launchd LaunchAgent plist with `StartCalendarInterval` + its maintenance script — morning start+warm+cockpit, daily health-check+doctor+notify, or daily benchmark log — + a `launchctl load` README) written to `~/Desktop/ollamas-daily/<model>/`. The scan then READS file content and `detectSchedule` verifies whether the artifact is actually recurring (launchd/cron/Calendar Alarm) → `orchestration/AUTOMATOR_DAILY.md` tracks "produced" AND "recurring". For that mode enable the daily write root too: add `$HOME/Desktop/ollamas-daily` to the `BRIDGE_WRITE_ROOTS` list above. Artifacts are produced + tracked, **never installed (`launchctl load`) or executed** — installing a recurring job is the operator's explicit decision. See `.claude/BRAIN.md`.
