Run `./node_modules/.bin/tsx orchestration/bin/chrome-probe.ts`. It reads the live model list (`ollama list`, embedding models skipped), then SEQUENTIALLY dispatches each model the task "open Google Chrome via `open -a \"Google Chrome\"` then confirm with pgrep" through `scripts/agent-dispatch.mjs` (POST /api/agent/chat on the live server, default `OLLAMAS_URL=http://127.0.0.1:3000`), and classifies each run: a model is **capable** only when it drove a shell tool successfully AND reached a DONE/OK verdict (not a demo/no-tool reply). Flags: `--dry` (print the plan without dispatching), `--models a,b` (subset), `--steps N` (default 4), `--json`. Reports which models opened Chrome. See `.claude/BRAIN.md`.
