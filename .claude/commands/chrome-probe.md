Run `./node_modules/.bin/tsx orchestration/bin/chrome-probe.ts`. It reads the live model list (`ollama list`, embedding models skipped), then SEQUENTIALLY dispatches each model the task "open Google Chrome via `open -a \"Google Chrome\"` then confirm with pgrep" through `scripts/agent-dispatch.mjs` (POST /api/agent/chat on the live server, default `OLLAMAS_URL=http://127.0.0.1:3000`), and classifies each run: a model is **capable** only when it drove a shell tool successfully AND reached a DONE/OK verdict (not a demo/no-tool reply). Flags: `--dry` (print the plan without dispatching), `--models a,b` (subset), `--steps N` (default 4), `--json`. Reports which models opened Chrome.

**`--task shortcuts`** (vO34): same sequential per-model dispatch + open-Chrome capability, but the task also asks each model to LIST the macOS Chrome dev/AI keyboard shortcuts (address bar, new tab, DevTools, JS console, inspect, reload…). Each answer is scored against a 14-combo canonical ground-truth (`CHROME_SHORTCUTS` in `bin/lib/chrome-probe.ts`) after normalizing ⌘/Cmd/Command etc. → writes `orchestration/CHROME_SHORTCUTS.md` (which models opened Chrome AND how many real shortcuts they named). The task only lists shortcuts + opens Chrome — no keystroke injection. See `.claude/BRAIN.md`.
