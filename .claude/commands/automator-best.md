Run `./node_modules/.bin/tsx orchestration/bin/automator-best.ts`. It reads `orchestration/AUTOMATOR_DAILY.json` (the output of `/automator-probe --loop`), ranks the recurring automations by completeness (a full launchd bundle = plist + maintenance script + README scores highest), then walks the ranking validating each candidate with `plutil -lint` (plist) + `bash -n` (script) — syntax only, never executed — and picks the highest-scoring one that PASSES. It copies that winner's files to `~/Desktop/ollamas-daily/BEST/` and writes an `INSTALL.md` with the one-command `launchctl load` install (+ uninstall). Reports which model won, its score, and the validation result. Flags: `--dry` (rank + validate without writing the bundle), `--from <json>` (alternate rows source). If AUTOMATOR_DAILY.json is missing, run `/automator-probe --loop` first. See `.claude/BRAIN.md`.
